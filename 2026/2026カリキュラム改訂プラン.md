## 大学Microsoft包括契約のCopilotについて

チュートリアル動画（`マイクロソフト包括契約でのAI等の活用_20260316.mp4`、約8分）から確認した内容：

### 使える機能

| 機能 | 使い方 |
|------|--------|
| **チャット（メイン）** | copilot.microsoft.com にアクセス → 「Microsoftで続行」→ 大学アカウントでサインイン |
| アイデア出し・翻訳 | チャット欄にテキスト入力 |
| 画像生成 | チャットで依頼 |
| ファイル添付・要約 | ファイルを添付してチャット |
| PowerPoint生成 | 「〜についてPPTXを作成して」と依頼 |
| **音声の文字起こし（トランスクリプト）** | Web版Word → ツール → 音声をアップロード ※OneDriveに保存される |

### アクセス方法（学生への案内）

1. ブラウザで `copilot.microsoft.com` を検索（または直接アクセス）
2. 「Microsoft で続行」を選択
3. 大学の Microsoft アカウント（大学メールアドレス）でサインイン
4. または Microsoft Store アプリから「copilot」で検索してアプリをインストール

### GitHub Copilot（VSCode内インライン補完）との違い

| | 大学契約の Microsoft Copilot | GitHub Copilot |
|-|------------------------------|---------------|
| 使い方 | ブラウザのチャット画面 | VSCode内でリアルタイム補完 |
| コードを渡す方法 | コピペしてチャットに貼る | エディタで書くと自動提案 |
| 取得方法 | 大学アカウントですぐ使える ✅ | 別途 GitHub Education Pack が必要 |
| 課題での利用 | 使いやすい・学生全員が使える ✅ | 設定が必要・個人差あり |

→ **授業では Microsoft Copilot（ブラウザ版）を標準ツールとして使う**

## Week 10 詳細：AI活用実践（1）基本操作と批判的評価

### 目標
- 大学Copilotへのサインイン方法を全員が習得する
- AIへの指示（プロンプト）の書き方を体験する
- AIの出力を**そのまま信頼せず検証する**姿勢を身につける

### 授業の流れ（90分）

**Step 1：セットアップ（15分）**

全員が `copilot.microsoft.com` にサインインできるか確認する。

```
1. ブラウザで copilot.microsoft.com を開く
2. 「Microsoft で続行」→ 大学アカウントでログイン
3. チャット画面が開くことを確認
```

**Step 2：AIにバグを探させる（30分）**

以下のバグ入りコードを Copilot に貼り付け、「このコードの問題を指摘してください」と入力する。
AIの説明を読んだあと、自分でも検証して正誤を確かめる。

```python
# 問題1：平均を計算するつもりが…
def calc_average(scores):
    total = 0
    for s in scores:
        total = total + s
    return total          # ← 何が足りない？

# 問題2：摂氏→華氏の変換
def celsius_to_fahrenheit(c):
    return c * 9 / 5 + 23  # ← 数値が間違っている

# 問題3：リストの偶数だけ取り出す
nums = [1, 2, 3, 4, 5, 6]
evens = [n for n in nums if n % 2 = 0]  # ← 構文エラー
```

**Step 3：自然言語→コード生成（30分）**

物理の問題を日本語でCopilotに渡し、Pythonコードを生成させる。
生成されたコードを実行し、物理的に正しいか自分で検証する。

> 「初速度 v0=10 m/s、角度 θ=45° で投げた物体の軌跡を
> matplotlib でプロットするPythonコードを書いてください。
> 空気抵抗はなし、g=9.8 m/s² とします。
> x軸・y軸にラベルを付け、グリッドを表示してください。」

→ 生成コードを実行 → 最高点や飛距離は計算で正しいか確認 → 必要なら修正

**Step 4：プロンプトを工夫する（15分）**

同じ問題に対して「良いプロンプト」と「悪いプロンプト」を比べる。

| 悪い例 | 良い例 |
|--------|--------|
| 「コード書いて」 | 「Pythonで、リストの要素数・最大・最小・合計を求めるコードを書いてください。for文は使わず、組み込み関数を使ってください」 |
| 「エラー直して」 | 「このPythonコードを実行すると `TypeError: unsupported operand type(s) for +: 'int' and 'str'` というエラーが出ます。原因と修正方法を教えてください：（コードを貼る）」 |

### 注意事項（学生への明示ルール）
- AIが生成したコードを**理解せずそのまま提出しない**
- AIは間違えることがある。必ず自分で**実行して検証**する
- レポート課題では「AIを使って生成した箇所」を**コメントで明記**する

## Week 13 詳細：AI活用実践（2）数値計算×AI

### 目標
- Week 12 で学んだ ODE（Runge-Kutta法）の知識を使い、AIと協力してシミュレーションを拡張する
- 「物理的な問いを立てる → AIにコードを作らせる → 結果を物理的に解釈する」サイクルを体験する

### 授業の流れ（90分）

**Step 1：Week 12 の復習コードを Copilot に渡す（20分）**

Week 12 で書いた惑星軌道のコードを Copilot に貼り付けて：
- 「このコードを説明してください」
- 「どこを変えると楕円の形が変わりますか？」

→ AIの説明を読み、実際にパラメータを変えて確かめる

**Step 2：AIに拡張を依頼する（40分）**

以下のいずれかを選んで、Copilotに依頼しながら実装する。

| テーマ | Copilotへの依頼例 |
|--------|-----------------|
| **空気抵抗ありの斜方投射** | 「抵抗係数 k を追加して、k=0 と k=0.1 の軌道を同じグラフに重ねてください」 |
| **惑星軌道の離心率** | 「初期速度を変えて離心率 e=0, 0.3, 0.6 の軌道を比較するグラフを作ってください」 |
| **ばねの減衰振動** | 「減衰係数 γ を変えて過減衰・臨界減衰・減衰振動の3通りをアニメーションで表示してください」 |

**Step 3：結果を物理的に解釈・発表（30分）**

各自が作成したグラフをスクリーンに映し、30秒〜1分で説明する。
- 「何のシミュレーションか」
- 「どのパラメータを変えたか」
- 「物理的にどんな意味があるか」

### AIとの協働パターン（授業で提示）

```
① 問いを立てる
   「空気抵抗があると飛距離はどう変わるか？」

       ↓

② AIにコードを依頼する
   「RK4を使って空気抵抗ありの斜方投射をシミュレートするコードを書いてください」

       ↓

③ 実行・エラー修正
   エラーが出たら → エラーメッセージをAIに貼り付けて「原因と修正方法は？」

       ↓

④ 物理的に検証
   k→0 の極限でk=0の解析解に近づくか確認

       ↓

⑤ 問いを発展させる
   「摩擦係数が大きすぎるとどうなるか？」→ ① に戻る
```

## 期末課題（Week 15）

### 出題方針
教員がテーマを**2〜3問指定**し、学生はそのうち**1問を選んで**実装・提出する。
2025年度のレポートNo.9（Lax法・Lax-Wendroff法）を拡張した形式。

### テーマ候補案

**問題A：数値積分と誤差解析**
台形則・Simpson則・ガウス求積のいずれかで定積分を数値的に計算し、
分割数 N を変えたときの誤差がどのように減少するかを両対数グラフで示せ。

**問題B：常微分方程式と解析解の比較**
RK4 を用いて以下のいずれかをシミュレートし、解析解（または既知の結果）と比較せよ。
- 減衰振動（減衰係数を変えて3通り以上）
- 太陽-惑星系の楕円軌道（ケプラーの第2法則を数値的に確認）

**問題C：偏微分方程式のシミュレーション**
Lax-Wendroff法と FTCS法（熱拡散）のどちらかを実装し、
初期条件を変えて物理的に意味のある結果を得よ。
数値安定性条件（CFL条件）についてもコメントせよ。

### 提出形式
- Jupyter Notebook（`.ipynb`）
- 学籍番号・氏名をコメントで記載
- AI を使用した箇所はコメントで明示（例：`# Copilot で生成、修正済み`）
- 提出期限：最終回（Week 15）の授業終了時
