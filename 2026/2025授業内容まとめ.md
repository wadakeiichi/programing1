## 開発環境

**構成：** VSCode + Docker + Dev Container（2025年度からの新方式）

### 必要ソフトウェア
- **VS Code**（https://code.visualstudio.com/）
- **Docker Desktop**（https://docs.docker.com/desktop/）
- **Git**（https://git-scm.com/）

### セットアップ手順（初回）

1. **Step 0：** CPU種別確認（x86-64 or ARM）
2. **Step 1：** VS Code インストール（Mac / Win11）
3. **Step 2：** Docker Desktop インストール
4. **Step 3：** Git インストール
   - Windows: git-scm.com からインストーラー実行
   - Mac: `brew install git`
5. **Step 4：** 授業用フォルダ作成・git clone
   ```bash
   cd ~/Documents
   mkdir programing2025
   cd programing2025
   git clone https://github.com/wadakeiichi/programing1.git
   ```
6. **Step 5-6：** VS Code でフォルダを開き、Docker Desktop を起動
7. **Step 7：** Dev Container を起動（初回は数分かかる）
   - 左下の `><` マーク →「Reopen in Container」
8. **Step 8：** `git pull` & `make` で環境を最新化

### Dev Container 構成
- **Dockerfile** に基づくカスタムコンテナ
- **postCreateCommand：** pip install + ipykernel セットアップ
- **カーネル名：** `Python (with numpy)`
- **主要拡張機能：** Python, Jupyter, 日本語パック, Vim

### 必要ライブラリ（requirements.txt）
```
jupyter
numpy
matplotlib
ipykernel
```

### 対応OS

- macOS 12以降

#### CUI基礎実習の内容案（Week 1〜2）

##### ▼ まず理解させること：「シェル」と「プロンプト」

CUIとは、コマンドを文字で入力してPCを操作する方式。
入力を受け付けるプログラムを**シェル**といい、Mac と Windows では種類が異なる。

| | Mac | Windows |
|--|-----|---------|
| シェルの種類 | **zsh**（macOS 10.15以降）または bash | **PowerShell** |
| ターミナルの開き方 | アプリ「ターミナル」または iTerm2 | スタートメニューで「PowerShell」検索 |
| プロンプトの見た目 | `keiichi@MacBook ~ %` または末尾が `$` | `PS C:\Users\keiichi>` |

**★重要：** 授業の資料やウェブで `$ cd Documents` と書いてあったら、
「`$` はプロンプト（入力待ちの印）であり、**入力しない**」。
実際に入力するのは `cd Documents` の部分だけ。
（PowerShellでも同様：`PS C:\Users\keiichi>` の `>` より右だけを入力する）

---

##### ▼ ディレクトリの概念

ファイルシステムは**木構造**（ツリー）になっている。「フォルダ」と「ディレクトリ」は同じ意味。

```
Mac のルート /                    Windows のルート C:\
└── Users/                        └── Users\
    └── keiichi/                      └── keiichi\
        ├── Documents/                    ├── Documents\
        │   └── programing2026/           │   └── programing2026\
        └── Desktop/                      └── Desktop\
```

- **ホームディレクトリ**：自分のユーザーフォルダ（`~` と書くと省略できる）
  - Mac: `/Users/keiichi`　→　`~` と書ける
  - Windows: `C:\Users\keiichi`　→　`~` と書ける（PowerShell 3以降）
- **カレントディレクトリ**：今いる場所。コマンドはここを基準に動く
- **パスの区切り文字**：Mac は `/`（スラッシュ）、Windows は `\`（バックスラッシュ）
  - ただし PowerShell は `/` も受け付けることが多い

**絶対パス vs 相対パス**

| | Mac の例 | Windows の例 |
|--|---------|-------------|
| 絶対パス | `/Users/keiichi/Documents/programing2026` | `C:\Users\keiichi\Documents\programing2026` |
| 相対パス（今が `Documents` なら） | `programing2026` または `./programing2026` | `programing2026` または `.\programing2026` |
| 一つ上 | `..` | `..` |
| ホームから | `~/Documents/programing2026` | `~\Documents\programing2026` |

---

##### ▼ 基本コマンド一覧

多くのコマンドは Mac と Windows (PowerShell) で**ほぼ共通**。
ただし**オプション（フラグ）の書き方が異なる**ので注意。

| 操作 | Mac (zsh) | Windows (PowerShell) | 備考 |
|------|-----------|----------------------|------|
| 現在地を表示 | `pwd` | `pwd` | 共通 |
| ファイル・フォルダ一覧 | `ls` | `ls` | 共通（内部コマンドは異なる） |
| 詳細一覧 | `ls -l` | `Get-ChildItem` | オプションが違う |
| ディレクトリ移動 | `cd フォルダ名` | `cd フォルダ名` | 共通 |
| 一つ上へ移動 | `cd ..` | `cd ..` | 共通 |
| ホームへ移動 | `cd ~` または `cd` | `cd ~` | 共通 |
| フォルダ作成 | `mkdir フォルダ名` | `mkdir フォルダ名` | 共通 |
| ネスト作成 | `mkdir -p a/b/c` | `mkdir a\b\c`（自動で作成） | オプションが違う |
| ファイルコピー | `cp 元 先` | `cp 元 先` | 共通 |
| ファイル移動・改名 | `mv 元 先` | `mv 元 先` | 共通 |
| ファイル削除 | `rm ファイル名` | `rm ファイル名` | 共通（ゴミ箱に入らない！） |
| テキスト表示 | `cat ファイル名` | `cat ファイル名` | 共通 |
| 画面クリア | `clear` | `cls` または `clear` | 少し違う |

> **注意（Windows）：** PowerShell の `ls` は `Get-ChildItem` のエイリアス。
> 動きは似ているが、`ls -la` のような Unix スタイルのオプションは使えない。
> オプションが必要なときは `Get-ChildItem -Force` などの PowerShell 流の書き方を使う。

##### ▼ 演習タスク案（Week 1）

Mac・Windows それぞれの画面を見せながら、**両者で同じ操作を並行してやってみる**。

1. ターミナル（Mac）または PowerShell（Windows）を開く
2. `pwd` で現在地を確認 → 「今どこにいるか」を言えるか
3. `ls` でファイル一覧を見る
4. `cd Documents` で移動 → `pwd` でパスが変わったことを確認
5. `mkdir programing2026` でフォルダを作成 → `ls` で確認
6. `cd programing2026` で移動 → `pwd` で確認
7. `cd ..` で戻る → `pwd` で確認
8. ホームディレクトリから `programing2026` への**絶対パス**を自分で書いてみる
