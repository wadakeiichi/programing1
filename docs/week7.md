### 文法解説

#### 制御構造とは ― プログラムの「流れ」を変える

これまでのプログラムは、書いた順に上から下へ一直線に実行されてきた。
**制御構造（control structure）** は、この流れを **条件によって枝分かれさせたり（分岐）**、**同じ処理を繰り返したり（ループ）** するための仕組みである。物理の計算では「条件に応じて式を変える」「収束するまで反復する」といった場面が頻出するので、制御構造は必須の道具になる。

この授業で扱う制御構造は次の 3 つ。

| 種類 | キーワード | 使う場面 |
|------|-----------|---------|
| 条件分岐 | `if` / `elif` / `else` | 値によって処理を変える（場合分け） |
| 回数指定ループ | `for` | 決まった回数・要素数だけ繰り返す（Week 4 で既出） |
| 条件ループ | `while` | 条件が成り立つ間ずっと繰り返す（収束計算など） |

---

#### 条件分岐 ― if / elif / else

ある条件が成り立つかどうかで処理を切り替えるのが `if` 文。条件が `True`（真）なら **インデントされたブロック** が実行される。

```python
v = 11.2   # 打ち上げ速度 [km/s]

if v >= 11.2:
    print("脱出速度を超えた → 地球を離れる")
else:
    print("地球周回軌道にとどまる")
```

複数の条件で分けたいときは `elif`（else if の略）を間に挟む。**上から順に判定され、最初に成り立った 1 つだけ** が実行される。

```python
T = 6000   # 表面温度 [K]

if T < 3500:
    spectral = "M 型（赤）"
elif T < 5000:
    spectral = "K 型（橙）"
elif T < 6000:
    spectral = "G 型（黄）"
else:
    spectral = "F 型以上（白〜青）"

print(f"温度 {T} K → {spectral}")
```

> **ポイント**: `if` / `elif` / `else` の行末には必ず **コロン `:`**、次の行は **インデント（半角スペース 4 つ）** が必要（Week 3b で学んだルール）。`else` には条件を書かない（「それ以外すべて」の意味）。

##### 比較演算子と論理演算子

条件の中で使う演算子をまとめておく。

| 比較演算子 | 意味 | | 論理演算子 | 意味 |
|-----------|------|--|-----------|------|
| `==` | 等しい | | `and` | かつ（両方が真） |
| `!=` | 等しくない | | `or` | または（どちらかが真） |
| `<` `<=` | 小なり・以下 | | `not` | 否定（真偽を反転） |
| `>` `>=` | 大なり・以上 | | | |

```python
m = 1.5   # 太陽質量単位

if m > 0.08 and m < 8:
    print("主系列星として安定に水素燃焼する")

if not (m > 8):
    print("超新星爆発は起こさない")
```

> **注意**: 「等しい」の比較は `==`（イコール 2 つ）。`=` 1 つは **代入** なので、条件に書くとエラーになる。よくある間違いなので気をつけること。

#### while ループ ― 条件が成り立つ間くりかえす

`for` は「決まった回数」を繰り返すのに対し、`while` は **条件が `True` の間ずっと** 繰り返す。回数が事前に分からない「収束するまで計算する」ような場面で使う。

```python
# 1 を 2 で割り続け、0.001 より小さくなるまで繰り返す
x = 1.0
count = 0
while x >= 0.001:
    x = x / 2
    count += 1
print(f"{count} 回で {x:.5f} になった")
```

`while` の条件が **いつまでも `True` のままだと無限ループ** になり、プログラムが止まらなくなる。必ず「ループの中で条件に関わる値が変化し、いつか条件が `False` になる」ように書くこと。

```python
n = 0
while n < 5:
    print(n)
    n += 1        # ← これを書き忘れると n はずっと 0 → 無限ループ！
```

> **無限ループに陥ったら**: VS Code のターミナル／インタラクティブウィンドウで実行が止まらなくなったら、`Ctrl + C` で強制停止できる。

##### break と continue

ループの途中で抜けたり、残りをスキップしたりする命令。

```python
for i in range(1000):
    if i ** 2 > 100:
        break        # 条件を満たしたらループを即終了
    print(i)

for i in range(10):
    if i % 2 == 0:
        continue     # 偶数のときは以降をスキップして次の i へ
    print(i)         # → 奇数だけ表示
```

- **`break`**: ループ全体を **その場で終了** する。
- **`continue`**: そのループの **残りをスキップ** して、次の繰り返しに進む。

#### 関数定義 ― def

同じ処理を何度も使うとき、**関数（function）** にまとめておくと「名前を呼ぶだけ」で再利用できる。`def` で定義する。

```python
def kinetic_energy(m, v):
    """質量 m [kg]、速さ v [m/s] の運動エネルギー [J] を返す"""
    E = 0.5 * m * v**2
    return E
```

- **`def 関数名(引数, ...):`** で定義を始める（行末コロン＋インデント）。
- **引数（ひきすう）**: 関数に渡す入力（ここでは `m`, `v`）。
- **`return`**: 計算結果を **呼び出し元に返す** 命令。`return` がないと `None` が返る。
- **`"""..."""`**: **docstring（説明文）**。関数が何をするかを 1 行目に書いておく習慣をつけると、後で読み返したときに分かりやすい。

定義した関数は次のように **呼び出す（call する）**。

```python
E1 = kinetic_energy(2.0, 3.0)     # m=2.0, v=3.0 を渡す
print(f"運動エネルギー: {E1} J")    # → 9.0 J

E2 = kinetic_energy(m=1.0, v=10.0)  # 引数名を明示してもよい（キーワード引数）
print(E2)                           # → 50.0
```

##### 複数の値を返す

`return` のあとにカンマで複数並べると、**複数の値をまとめて返せる**（タプルとして返る）。

```python
def projectile_range(v0, theta_deg, g=9.81):
    """斜方投射の飛距離と最高到達点を返す"""
    theta = np.radians(theta_deg)
    T = 2 * v0 * np.sin(theta) / g
    R = v0 * np.cos(theta) * T          # 飛距離
    H = (v0 * np.sin(theta))**2 / (2*g)  # 最高点
    return R, H

distance, height = projectile_range(20.0, 45)   # 2 つの戻り値を受け取る
print(f"飛距離 {distance:.1f} m, 最高点 {height:.1f} m")
```

##### デフォルト引数

引数に `= 値` を付けておくと、**呼び出し時に省略したときその値が使われる**。上の `g=9.81` がそれ。

```python
projectile_range(20.0, 45)          # g は 9.81（地球）が使われる
projectile_range(20.0, 45, g=1.62)  # 月の重力を指定して計算
```

> **ポイント**: デフォルト引数は「ふだんは決まった値だが、たまに変えたい」パラメータに便利。物理定数や設定値によく使う。引数の並びでは、**デフォルト付きの引数は必ず後ろ** に置くこと（`def f(a, b=1)` は OK、`def f(a=1, b)` はエラー）。

#### 自作モジュールの import ― 自分の関数を別ファイルから使う

Week 4 で `import math` のように既製のモジュールを読み込んだ。実は **自分で書いた `.py` ファイルもモジュールとして `import` できる**。よく使う関数を 1 つのファイルにまとめておくと、いろいろなプログラムから再利用できる。

たとえば、物理の関数を集めた `physics_tools.py` を作るとする。

```python
# physics_tools.py  ← 関数だけを書いておくファイル
import numpy as np

def kinetic_energy(m, v):
    """運動エネルギー [J]"""
    return 0.5 * m * v**2

def escape_velocity(M, R):
    """質量 M [kg]、半径 R [m] の天体の脱出速度 [m/s]"""
    G = 6.674e-11
    return np.sqrt(2 * G * M / R)
```

同じフォルダにある別のファイルから、**ファイル名（拡張子 `.py` を除く）** で `import` して使える。

```python
# week7.py など、使う側のファイル
import physics_tools as pt          # physics_tools.py を読み込む

print(pt.kinetic_energy(2.0, 3.0))                 # → 9.0
print(pt.escape_velocity(5.972e24, 6.371e6))       # 地球の脱出速度 ≈ 11186 m/s
```

特定の関数だけ読み込みたいときは `from` を使う（Week 4 の `from math import sqrt` と同じ書き方）。

```python
from physics_tools import escape_velocity
print(escape_velocity(5.972e24, 6.371e6))
```

> **ポイント**: 自作モジュールは **import する側と同じフォルダ** に置くこと。関数をまとめておくと、コードの見通しがよくなり、同じ関数を何度も書かずに済む（再利用）。これは大きなプログラムを書くときの基本になる。

### 例 1 ― ニュートン法で平方根を求める（while による収束計算）

「収束するまで繰り返す」典型例として、ニュートン法で √a を求める。漸化式 `x ← (x + a/x) / 2` を、変化量が十分小さくなるまで `while` で反復する。

```python
# week7_ex1.py
def my_sqrt(a, tol=1e-10, max_iter=100):
    """ニュートン法で a の平方根を求める。
    tol: 収束判定のしきい値, max_iter: 最大反復回数"""
    if a < 0:
        print("負の数の平方根は実数では求められない")
        return None

    x = a          # 初期値
    for i in range(max_iter):
        x_new = 0.5 * (x + a / x)     # 漸化式
        if abs(x_new - x) < tol:      # 変化が十分小さければ収束
            x = x_new
            break
        x = x_new

    return x

# いくつかの値で試し、math.sqrt と比較する
import math
for a in [2.0, 10.0, 100.0]:
    approx = my_sqrt(a)
    print(f"√{a:6.1f} = {approx:.10f}  (math: {math.sqrt(a):.10f})")
```

**実行結果の確認ポイント**: 自作の `my_sqrt` の値が `math.sqrt` とほぼ一致すれば正解。`for` の中で収束したら `break` で抜けている点に注目。

### 例 2 ― 自作モジュールにまとめて再利用する

物理の関数を `physics_tools.py` にまとめ、`week7_ex2.py` から `import` して使う。**2 つのファイルを同じフォルダに作る** こと。

```python
# physics_tools.py
import numpy as np

def escape_velocity(M, R):
    """質量 M [kg]、半径 R [m] の天体の脱出速度 [m/s]"""
    G = 6.674e-11
    return np.sqrt(2 * G * M / R)

def classify_star(T):
    """表面温度 T [K] からスペクトル型をざっくり分類して返す"""
    if T < 3500:
        return "M"
    elif T < 5000:
        return "K"
    elif T < 6000:
        return "G"
    elif T < 7500:
        return "F"
    else:
        return "A 以上"
```

```python
# week7_ex2.py
import physics_tools as pt

# 太陽系の天体（名前, 質量[kg], 半径[m]）
bodies = [
    ("地球", 5.972e24, 6.371e6),
    ("月",   7.342e22, 1.737e6),
    ("木星", 1.898e27, 6.991e7),
]

print("=== 脱出速度 ===")
for name, M, R in bodies:
    v = pt.escape_velocity(M, R)
    print(f"{name}: {v/1000:.2f} km/s")

print("\n=== スペクトル分類 ===")
for T in [3000, 5800, 9000]:
    print(f"T = {T:5d} K → {pt.classify_star(T)} 型")
```

**実行結果の確認ポイント**: 地球の脱出速度が約 11.2 km/s、太陽（≈5800 K）が G 型と表示されれば正解。関数を別ファイルにまとめ、`pt.関数名()` で呼べていることを確認する。

### 練習問題 4

以下の練習問題を `programing1/week7/` に `week7_practice4.py`（および指定があるモジュールファイル）として保存せよ。各問題は `# %%` で区切ること。

3. **自作モジュール**: 運動エネルギー `kinetic_energy(m, v)`、脱出速度 `escape_velocity(M, R)`（デフォルト引数で `G=6.674e-11`）、斜方投射の飛距離 `projectile_range(v0, theta_deg, g=9.81)` の 3 つの関数を `physics_tools.py` にまとめよ。各関数には docstring を付けること。別ファイル `week7_practice4.py` から `import physics_tools as pt` で読み込み、地球と月（`g=1.62`）での斜方投射の飛距離を計算して比較せよ。
