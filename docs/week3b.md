### 文法解説

#### 1. コロン `:` ― 「ここからブロックが始まるよ」の合図

Python では `if`、`for`、`while`、`def` などの行の末尾に **必ずコロン `:`** を書く。
コロンは「この下にまとまった処理が続きますよ」という合図。

```python
# ✅ 正しい（コロンあり）
if x > 0:
    print("正の数")

# ❌ エラー（コロンがない）
if x > 0
    print("正の数")
#   → SyntaxError: expected ':'
```

コロンが必要になる場面はこの 4 つ：

| 構文 | 書き方 |
|------|--------|
| 条件分岐 | `if 条件:` / `elif 条件:` / `else:` |
| 繰り返し | `for 変数 in ...:` / `while 条件:` |
| 関数定義 | `def 関数名(引数):` |
| クラス定義 | `class クラス名:` （この授業では使わない） |

> **覚え方**: 「何かの中身を書く前にはコロン」。日本語の「：」と同じ感覚。

#### 2. インデント ― Python 最大の特徴

**インデント（行頭の空白）**は Python では**文法の一部**。
C言語の `{ }` や Fortran の `END IF` の代わりに、**スペースの深さでブロック（まとまり）を表す**。

```python
# ✅ 正しいインデント（半角スペース 4 つ）
x = 10
if x > 0:
    print("正の数")    # ← スペース 4 つ分ずれている = if の中身
    print("x =", x)    # ← これも if の中身
print("終わり")         # ← インデントなし = if の外

# 実行結果:
#   正の数
#   x = 10
#   終わり
```

```python
# ❌ インデントがない → エラー
if x > 0:
print("正の数")
# → IndentationError: expected an indented block after 'if' statement
```

```python
# ❌ インデントがバラバラ → エラー
if x > 0:
    print("1行目")      # スペース 4 つ
      print("2行目")    # スペース 6 つ（不一致！）
# → IndentationError: unexpected indent
```

**ルール**:
- コロン `:` の次の行から、**半角スペース 4 つ**でインデントする
- 同じブロック内のインデントは**すべて同じ深さ**にそろえる
- **タブ文字は使わない**（スペースとタブが混在するとエラーの原因になる）
- VS Code では Enter を押すと自動でスペース 4 つが入るので、基本的にはそのままで OK

##### インデントの「深さ」が意味を変える例

```python
for i in range(3):
    print(i)
    print("---")       # ← for の中なので 3 回表示される

# 実行結果:
#   0
#   ---
#   1
#   ---
#   2
#   ---
```

```python
for i in range(3):
    print(i)
print("---")           # ← for の外なので 1 回だけ表示される

# 実行結果:
#   0
#   1
#   2
#   ---
```

この 2 つのコードは `print("---")` のインデント（中か外か）が違うだけで、結果がまったく異なる。

#### 3. 空白（スペース）の「ある・なし」

##### 演算子の前後のスペース ― あってもなくても動くが、読みやすさが違う

```python
# どちらも同じ結果だが…
y=2*x+1       # ❌ 読みにくい
y = 2 * x + 1  # ✅ 読みやすい（推奨）
```

Python の公式スタイルガイド（PEP 8）では **演算子の前後にスペース 1 つ** を推奨している。
この授業でも、読みやすさのために演算子の前後にはスペースを入れること。

##### `=`（代入）と `==`（比較）の違い

```python
x = 5       # 代入: x に 5 を入れる
x == 5      # 比較: x が 5 と等しいか？ → True または False
```

初心者が最もよくやるミスの一つ。`=` は「入れる」、`==` は「同じ？」と覚えよう。

##### 関数呼び出しの括弧の前 ― スペースを入れない

```python
# ✅ 正しい
print("hello")
math.sqrt(2)
np.linspace(0, 1, 100)

# ❌ スペースが入っている（動くがスタイル違反）
print ("hello")
math.sqrt (2)
```

##### カンマの後にスペース ― 入れる

```python
# ✅ 読みやすい
np.linspace(0, 1, 100)
data = [1, 2, 3, 4, 5]

# ❌ 読みにくい
np.linspace(0,1,100)
data = [1,2,3,4,5]
```

##### スペースが意味を変えるケース ― マイナス符号

```python
x = 10
print(x - 3)    # 引き算   → 7
print(x, -3)    # 2 つの値の表示 → 10 -3
```

「`x - 3`」は引き算、「`x, -3`」はカンマ区切りで 2 つの値を print に渡している。
書き方で意味が変わるので注意。

#### 4. コメント `#` ― メモを残す

`#` より右側はすべてコメント（Python は無視する）。
**なぜその計算をしているのか**を書いておくと、後から見直したとき（そして採点するとき）助かる。

```python
c = 3.0e8           # 光速 [m/s]
m_e = 9.109e-31     # 電子の質量 [kg]
E = m_e * c**2      # 静止エネルギー E = mc²
```

> **ポイント**: 「何をしているか」よりも「なぜ・何の物理量か」を書くと良いコメントになる。
> `# 掛け算する` ではなく `# 静止エネルギー E = mc²` のように。

#### 5. セル区切り `# %%` ― VS Code でファイルを「区切って実行」する

`# %%` は **VS Code（Python 拡張＋Jupyter 拡張）に対する目印** で、`.py` ファイルを
**Jupyter ノートブックのようにセル単位で実行できる**ようにする特別なコメント。
中身は通常の `#` コメントなので、Python から見ればただの何もしない行として扱われる
（ターミナルで `python file.py` と実行してもそのまま動く）。

```python
# %% セル1: 変数の準備
x = 10
y = 20

# %% セル2: 計算してみる
print(x + y)        # ← セル1 で作った x, y をそのまま使える

# %% セル3: グラフを描く例
import matplotlib.pyplot as plt
plt.plot([1, 2, 3], [1, 4, 9])
plt.show()
```

**書き方のルール**:

- 行頭から `# %%` と書く（`#` と `%%` の間にスペースを 1 つ入れる）
- `# %%` の右側にはセルのタイトルやメモを自由に書ける（例: `# %% セル1: 変数の準備`）
- `# %%` を書いた行から、次の `# %%` の手前までが **1 セル**
- ファイルの一番上にも `# %%` を置けば、最初のセルになる

**使い方（VS Code）**:

- セル（`# %%` から次の `# %%` までの範囲）の上に出る **「Run Cell」** をクリックする、
  または **`Shift + Enter`** でセルを実行
- 1 つ目のセルを実行すると右側に「Python Interactive Window」が開き、結果はそこに表示される
- 変数（上の例の `x`, `y`）はセルをまたいで保持される。
  `x + y` を別のセルで値を変えながら何度でも計算し直せる

**何が嬉しいか**:

- ファイル全体を毎回実行し直さずに、**「変えた部分だけ」をすぐ試せる**
- Jupyter ノートブック (`.ipynb`) と違って **ファイルは普通の `.py`** なので、
  Git で差分が見やすく、`python file.py` でも動く
- 練習問題やレポート作成で「試行錯誤しながら書く」場面に向いている

> **覚え方**: `# %%` は「ここからセルを区切る」マーカー。書かなくてもプログラムは動くが、
> この授業の例題（`# %% セル1` などが出てくるもの）は **セル単位で実行する前提** で書かれている。
> 例題を写すときは `# %%` の行も省略せずに入れておくこと。

### 例 1 ― インデントの違いで結果が変わることを体験する

```python
# week3b_ex1.py

# %% セル1: for の中と外
print("=== パターン A ===")
for i in range(3):
    print(f"  i = {i}")
    print(f"  i² = {i**2}")
print("ループ終了")    # ← for の外

print()

print("=== パターン B ===")
for i in range(3):
    print(f"  i = {i}")
print(f"  i² = {i**2}")   # ← for の外（最後の i の値だけ使われる）
print("ループ終了")
```

**実行結果の確認ポイント**: パターン A では `i²` が 3 回（0, 1, 4）表示されるが、パターン B では 1 回（4 だけ）しか表示されない。`print(f"  i² = {i**2}")` のインデントの違いだけでこの差が生まれる。

### 例 2 ― よくあるエラーとその直し方

```python
# week3b_ex2.py

# %% セル1: コロン忘れ（実行するとエラーになる → 直して再実行）
x = 5
if x > 0
    print("正")

# %% セル2: インデント不一致（実行するとエラーになる → 直して再実行）
for i in range(3):
    print(i)
      print(i * 2)

# %% セル3: = と == の混同
a = 10
b = 10
print(a = b)     # ← これはエラーになる。正しくは a == b

# %% セル4: 正しく直したバージョン
x = 5
if x > 0:                 # コロンを追加
    print("正")

for i in range(3):
    print(i)
    print(i * 2)           # インデントをそろえた

a = 10
b = 10
print(a == b)              # == に修正 → True
```

**実行結果の確認ポイント**: セル1〜3 はわざとエラーが出るコード。エラーメッセージを読んで、何が問題かを考えてからセル4 の修正版と見比べること。
