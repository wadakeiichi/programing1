### 文法解説

#### 配列とは ― 数学のベクトル・行列との関係

物理数学で扱う**ベクトル**や**行列**は、プログラミングでは**配列（array）**として表現する。
似ているところと違うところがあるので、最初に整理しておこう。

##### 数学のベクトル・行列

```
1次元（ベクトル）:  v = (1.0,  2.0,  3.0)          ← 3つの成分を持つ
2次元（行列）:      A = | 1  2  3 |                ← 2行3列の行列
                      | 4  5  6 |
```

数学では「ベクトルの内積」「行列の固有値」など、線形代数のルールに従って演算する。

##### プログラムの配列

プログラムの配列は、**数値を順番に並べて格納する入れ物**。
数学のベクトル・行列を表現できるが、それだけでなく「1000 個の測定データ」「画像の全ピクセル値」のように、**大量のデータをまとめて扱う**ための汎用的な道具。

```
1次元配列:  [1.0, 2.0, 3.0]           ← ベクトルにも、時系列データにも使える
2次元配列:  [[1, 2, 3],               ← 行列にも、画像データにも使える
            [4, 5, 6]]
```

##### なぜ NumPy の配列を使うのか

Python のリスト（Week 4 で学んだ `[ ]`）でもデータは格納できるが、
NumPy の配列（`np.array`）には物理計算で決定的に重要な利点がある:

| | Python リスト | NumPy 配列 |
|--|--------------|------------|
| `a * 2` の意味 | リストを 2 回繰り返す | **全要素を 2 倍**（数学的） |
| `a + b` の意味 | リストを連結 | **要素ごとの足し算**（ベクトル加算） |
| 計算速度 | 遅い（for ループが必要） | **50〜200 倍速い**（例 1 で体験） |
| 数学関数 | 1 つずつ計算 | `np.sin(a)` で**全要素を一括計算** |

```python
import numpy as np

# Python リストの場合
a_list = [1, 2, 3]
a_list * 2              # → [1, 2, 3, 1, 2, 3]  （繰り返し！）

# NumPy 配列の場合
a_np = np.array([1, 2, 3])
a_np * 2                # → [2, 4, 6]            （全要素 ×2！）
```

> **まとめ**: NumPy の配列は「数学のベクトル・行列のように要素ごとの演算ができて、
> しかも大量のデータを高速に処理できる入れ物」と考えればよい。
> この授業では、物理量の計算はすべて NumPy 配列で行う。

#### 配列の作成
```python
import numpy as np

a = np.array([1.0, 2.0, 3.0])           # リストから作成
t = np.linspace(0, 1, 5)                 # 0〜1 を 5 等分 → [0, 0.25, 0.5, 0.75, 1.0]
x = np.zeros(4)                           # ゼロ配列
y = np.ones(4)                            # 1 の配列
z = np.arange(0, 10, 2)                  # [0, 2, 4, 6, 8]
```

#### 配列の演算（ブロードキャスト）
```python
a = np.array([1.0, 2.0, 3.0])

a * 2          # → [2.0, 4.0, 6.0]   全要素に ×2
a + 10         # → [11., 12., 13.]   全要素に +10
a ** 2         # → [1., 4., 9.]      全要素を 2 乗
np.sqrt(a)     # → [1., 1.41, 1.73] 全要素の平方根
np.sin(a)      # 全要素の sin（ラジアン）
```

#### 集計・検索
```python
a = np.array([3.0, 1.0, 4.0, 1.0, 5.0])
np.max(a)       # 最大値: 5.0
np.min(a)       # 最小値: 1.0
np.argmax(a)    # 最大値のインデックス: 4
np.mean(a)      # 平均: 2.8
np.sum(a)       # 合計: 14.0
```

### 例 1 ― for ループと NumPy の速度比較

```python
# week5_ex1.py
import numpy as np
import time

N = 1_000_000   # 100 万点

# ① for ループ版
t_arr = [i / N for i in range(N)]   # Python リスト
start = time.time()
y_loop = [2.0 * (i / N) ** 2 for i in range(N)]
t_loop = time.time() - start
print(f"for ループ: {t_loop:.3f} 秒")

# ② NumPy 版
t_np = np.linspace(0, 1, N)
start = time.time()
y_np = 2.0 * t_np ** 2
t_numpy = time.time() - start
print(f"NumPy    : {t_numpy:.3f} 秒")

print(f"速度比   : {t_loop / t_numpy:.1f} 倍")
```

**実行結果の確認ポイント**: NumPy は for ループより 50〜200 倍程度速い。

### 例 2 ― プランク黒体放射のピーク波長

```python
# week5_ex2.py
import numpy as np

# 物理定数
h = 6.626e-34   # プランク定数 [J·s]
c = 3.0e8       # 光速 [m/s]
k = 1.381e-23   # ボルツマン定数 [J/K]

# 波長配列（nm → m に変換）
lam_nm = np.linspace(100, 3000, 5000)   # [nm]
lam = lam_nm * 1e-9                      # [m]

for T in [3000, 6000, 15000]:   # K
    B = (2 * h * c**2 / lam**5) / (np.exp(h * c / (lam * k * T)) - 1)
    peak_nm = lam_nm[np.argmax(B)]
    wien = 2.898e6 / T   # ウィーンの変位則 [nm]
    print(f"T = {T:6d} K :  ピーク = {peak_nm:.1f} nm  "
          f"（ウィーン則 = {wien:.1f} nm）")
```

**実行結果の確認ポイント**: 数値解がウィーンの変位則とほぼ一致すれば正解。
