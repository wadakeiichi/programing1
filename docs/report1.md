## 1. ファイルの書き方ルール（必ず守ること）

### 1.1 `# %%` でセルを区切る

ファイル全体を **1 つの大きな `.py`** にする。問題ごとに `# %%` でセルを区切り、VS Code の「Run Cell」（または `Shift + Enter`）で **1 問ずつ実行できる** ようにすること。

```python
# %% 問題1: タイトル
# （ここに問題1のコードを書く）

# %% 問題2: タイトル
# （ここに問題2のコードを書く）
```

### 1.2 ファイル冒頭にヘッダコメントを書く

ファイルの一番上に、次の情報をコメント（`#`）で書く。

```python
# ============================================
# ミニレポート 1
# 氏名: 〇〇 〇〇
# 学籍番号: XXXXXXXX
# 提出日: 2026-MM-DD
# ============================================
```

### 1.3 コードを書くときの作法（Week 3b で学んだこと）

- `if` / `for` / `def` の行末には **必ずコロン `:`** を書く。
- ブロックの中は **半角スペース 4 つ** でインデント。タブ文字は使わない。
- 演算子（`+` `-` `*` `/` `=` など）の **前後にはスペース 1 つ**（PEP 8）。
- **コメントには「なぜその計算をしているか／何の物理量か」を書く**。例: `c = 3.0e8   # 光速 [m/s]`
- 数値リテラルには **単位** をコメントで添える。

### 問題 1 ― 水素原子の基本物理量

水素原子の **基底状態のエネルギー** と **ボーア半径** を、定義式から計算して表示せよ。

定義式:

- ボーア半径: $a_0 = \dfrac{\hbar^2}{m_e\,e^2\,k_e}$
- 基底状態エネルギー: $E_1 = -\dfrac{m_e\,e^4\,k_e^2}{2\,\hbar^2}$

使用する物理定数（**変数として宣言し、コメントで単位を書くこと**）:

| 記号 | 値 | 単位 |
|------|------|------|
| $\hbar$ | $1.0546 \times 10^{-34}$ | J·s |
| $m_e$ | $9.109 \times 10^{-31}$ | kg |
| $e$ | $1.602 \times 10^{-19}$ | C |
| $k_e$ | $8.988 \times 10^{9}$ | N·m²/C² |
| $1\,\mathrm{eV}$ | $1.602 \times 10^{-19}$ | J |

**要件**:

1. 各定数を変数に代入し、`a0` と `E1` を計算する。
2. `a0` を **メートル** と **オングストローム**（$1\,\text{\AA} = 10^{-10}$ m）で表示。
3. `E1` を **ジュール** と **eV** の両方で表示。
4. f-string の書式指定子を使い、**指数表記 4 桁** または **小数 4 桁** で揃えて出力する。

**期待される結果（おおよそ）**: $a_0 \approx 5.29 \times 10^{-11}$ m $\approx 0.529$ Å、$E_1 \approx -13.6$ eV

### 問題 2 ― バグ修正と浮動小数点の比較

#### (a) バグ修正

以下のコードには **Week 3b で学んだ書き方のルール違反** が複数（少なくとも 3 つ）含まれている。コードをセルにコピーしてエラーを再現したあと、**正しいコードに直して** 同じセルの下に書き直せ。何を直したかをコメントで明記すること。

```python
# わざと壊してあるコード（このまま実行するとエラー）
total = 0
for i in range(1, 6)
    total = total + i ** 2
  print(f"途中経過: total = {total}")
print(f'最終結果: 1^2 + 2^2 + ... + 5^2 = {total}')
print(total = 55)
```

#### (b) 浮動小数点の比較

次の出力を実行し、**結果と理由をコメントで説明** せよ。

```python
a = 0.1 + 0.2
b = 0.3
print(a == b)
print(f"a = {a:.20f}")
print(f"b = {b:.20f}")
```

ヒント: コンピュータは数を **2 進法で有限ビット** に格納するため、10 進法の `0.1` は厳密には表現できない。なぜ `==` での比較を避けるべきか、3 行程度のコメントで自分の言葉で書くこと。
